# Video to Prompt

영상 → 프레임 캡처 → 시각 분석 → 영상 생성 모델용 프롬프트. API 호출 없이 세션 안에서 직접 프레임을 읽어 분석한다.

## 절차

### 1. 입력 확인
- 영상 파일 경로를 확인한다. 없으면 사용자에게 요청.
- 타겟 모델을 확인한다 (Higgsfield / Kling / Runway). 지정이 없으면 **3개 모두** 생성한다.

### 2. 프레임 추출
임시 작업 디렉토리(스크래치패드 또는 시스템 temp)에 출력 디렉토리를 만들고 추출 스크립트를 실행한다:

```bash
bash <skill_dir>/scripts/extract_frames.sh <video> <temp_dir>/frames [max_frames]
```

- 기본 1fps, 최대 60프레임 (긴 영상은 간격이 자동으로 넓어짐).
- 스크립트가 출력하는 `manifest.txt`로 각 프레임의 타임스탬프를 확인한다.
- ffmpeg 미설치 에러가 나면 `brew install ffmpeg` 안내 후 설치.

### 3. 프레임 분석
- 이미지 뷰어 도구(Claude Code는 Read, Codex는 view_image)로 프레임을 순서대로 읽는다. **한 번에 10~15장씩 배치**로 처리.
- 각 프레임을 타임스탬프와 연결해 다음을 관찰한다:
  - 피사체: 인물/사물의 외형, 의상, 표정, 위치
  - 동작: 프레임 간 변화로 움직임 추론
  - 카메라: 구도 변화로 카메라 워크 추론 (dolly, pan, orbit, zoom, handheld, static)
  - 샷 사이즈: wide / medium / close-up과 그 전환
  - 조명·색감: 광원 방향, 색온도, 그레이딩, 대비
  - 씬 경계: 배경·구도가 급변하는 지점 = 컷
- 프레임이 60장을 넘거나 특정 구간이 애매하면, 스크립트를 `start_sec`/`duration_sec` 인자로 재실행해 그 구간만 촘촘히 다시 추출한다.
- 오디오가 중요해 보이는 영상(대사/내레이션)이면 그 한계를 결과에 명시한다 (현재 화면 분석만 수행).

### 4. 프롬프트 생성
`<skill_dir>/references/prompt-guides.md`를 읽고, 타겟 모델의 문법에 맞춰 생성한다:

1. **영상 분석 요약** (한국어) — 씬별 타임스탬프/내용/카메라/조명 표
2. **Master Prompt** (영어) — 모델별 1개
3. **Scene-by-scene Prompts** (영어) — 컷이 2개 이상이면 씬별로
4. 메타: 종횡비, 원본 길이, 추천 생성 설정

### 5. 검증 (선택)
사용자가 원하면 대표 프레임과 생성 프롬프트를 나란히 보여주고, 빠진 시각 요소가 없는지 자체 점검 후 프롬프트를 1회 다듬는다.

## 주의
- 프레임에서 실제로 관찰한 것만 프롬프트에 넣는다. 브랜드명/실존 인물명은 일반화한다.
- 임시 파일(프레임)은 임시 작업 디렉토리에만 생성한다.
